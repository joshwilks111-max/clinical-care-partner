★The one-sentence thesis

Heidi Evidence retrieves and cites the guideline — that's one step inside this flow. This is the care-partner layer around it: build the differential, let the clinician steer, then execute safely. The moat is the differential reasoning about absent evidence + the deterministic safety spine — the judgment retrieval doesn't do.

Judgment up, execution down. The LLM does the thinking (differential, choosing which dose rule). Everything that can hurt a patient — picking the guideline, doing the arithmetic — is deterministic and auditable. That boundary is the whole design, and it must be a visible seam in the UI, not just an architectural claim.
Interface: structured console, not a chatbot (locked via /design-shotgun — see mockups/). Two-panel workspace: LEFT = case + confirm-weight, RIGHT = stepped Turn 1 → Turn 2 cards. The console keeps the safety spine persistent (a chat thread scrolls it away). A chatbot is a viable future interface, not foreclosed — it's the headline "if I had another day" Loom beat.
Reuse vs write (verified via /last30days): WRITE the differential + safety spine (the moat). REUSE the plumbing — shadcn/ui base + Vercel AI Elements leaf components (Tool, InlineCitation, Sources, PromptInput) — but NOT its Conversation shell. Eval UI = Promptfoo's built-in promptfoo view. Re-verify component names at the spike.
LLM judgment — model-generated DET deterministic — tool / registry / code SAFETY deliberate safety event — amber, never red

1Architecture flow

Two native round-trips with a human-in-the-loop STOP between them. A server-owned CaseState carries turn 1's confirmed outputs into turn 2 — turn 2 does zero re-extraction, so each turn is independently reproducible.

clinician note untrusted — wrapped in "treat as data" delimiters

— TURN 1 · DIFFERENTIAL LLM —
  → extract facts + weighted differential (positive / negative evidence)
  → DET PRE-LLM gate: weight == null? → REFUSE, no model call
  → render differential + guideline buttons + confirm weight → stream STOPS
  → clinician selects guideline + confirms weight
═══ judgment ends · deterministic execution begins ═══ — TURN 2 · APPLY DET / CONSTRAINED —
  → DET router: (condition, profession, setting) → guideline_id  logged + audited vs confirmed condition
  → DET get_guideline(id) → whole doc + DoseRule + RequiredFields
  → LLM classify severity vs the guideline's table → pick dose_rule_id (rule-application, bounded)
  → DET calculate_dose(guideline_id, dose_rule_id, weight_kg) → {dose_mg, dose_ml, capped...} LLM never does math
  → LLM synthesise plan, cite verbatim (Zod-constrained; each recommendation carries its citation)
  → DET completeness gate: every RequiredFields slot present AND non-null?
  → DET refusal gate: weight missing OR no matching guideline → ABSTAIN
The dose tool looks the rule up by id from the committed registry — the LLM picks which rule (severity row), never what it says. A hallucinated or injected note structurally cannot change the cap. That closes the gap between asserting the trust boundary and enforcing it.

2UI states — plain, but with hierarchy

3Code contracts (the safety spine)

The dose tool — registry owns every number

// LLM picks WHICH rule (by id). Never what the rule says.
function calculate_dose(
  guideline_id: string,
  dose_rule_id: string,
  weight_kg:    number
): {
  dose_mg: number; dose_ml: number | null;
  calculation_trace: string;
  capped: boolean; binding_limit: number | null;
  data_gaps: string[];
}
// tool reads drug/mg_per_kg/max_mg/route/
// concentration/rounding/min_mg FROM the registry.
// rejects an invalid dose_rule_id.

DoseRule — the committed, version-pinned rule

type DoseRule = {
  dose_rule_id: string;
  drug: string; mg_per_kg: number;
  min_mg: number|null; max_mg: number;
  route: string; frequency: string;
  concentration_mg_per_ml: number|null; // mg→mL det.
  rounding: { direction:"down"|"nearest";
              increment_mg:number }|null; // GUARD-8
  source_section; source_version; source_url;
  human_verified: boolean; // gates execution
};

Structured output — makes the completeness gate real

type PlanOutput = {
  recommendations: Array<{
    text; source_section; source_version;
    source_url; quote;        // citation enforced
  }>;
  required_fields: Record<string, {
    present: boolean;
    value: string | null;  // gate: present AND non-null
  }>;                          // "Escalation: not specified" FAILS
};

CaseState — the turn-1 → turn-2 contract

type CaseState = {  // server-owned, signed
  note_hash: string;
  extracted_facts: ExtractedFacts;
  differential: Differential;
  selected_condition: string;
  selected_guideline_id: string;
  selected_severity: string;
};
// turn 2 consumes this verbatim — zero re-extraction.
Two test layers. Promptfoo (6 demo cases + injection + no-guideline) exercises the LLM behaviours, asserting against structured output not prose. Unit tests (tools/*.test.ts) exercise the deterministic guards/edges (exactly-at-cap, zero/negative weight, GUARD-8 rounding, min_mg floor, Zod-fail). Run with npm run eval → named checks, not aggregate.

6Deliberately deferred — the depth signal

Named on purpose, each with a build-trigger. The deferred list IS the senior signal for a 4-day take-home. Full table in TODOS.md.

Wrong-guideline auto-abstain
Audit assertion ships; auto-abstain behaviour deferred. The nastier failure than empty-context.
Differential-collapse loop
Ambiguous note → discriminating question → narrow dx. The real care-partner product. Lead the "another day" beat.
Model-tier routing
Big model judgment / light model execution. Cost-opt, unneeded at demo scale.
Deterministic severity mapping
Encode severity criteria as typed rules. Regulatory hardening; brittle on free-text for a PoC.
LLM-as-judge (Layer 2)
Clinically-framed applicability rubric. Hook in tests/evals; informational, never gating.
Live-consult + scale retrieval + privacy
Transcription, agentic retrieval over a large corpus, on-prem/no-PHI. The 6-month product.
